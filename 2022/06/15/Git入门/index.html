简化版提交操作

git commit后会紧跟着打开vim终端编辑器

  • 按a或i进入编辑模式
  • 键入说明字符串
  • 按esc退出编辑模式
  • :wq保存并退出
    注意：
  • git只有两种修改类型：新增，删除
  • 规范化提交备注，如fix(文件名/模块): 修改内容（搜索git commit风格）
  • vscode source control可以自动实现基础操作（高级功能可以安装git history diff插件）
    查看提交信息 回退到某个版本（用到版本id或者id的前几位就足够了）
  • hard是硬重置，另有soft，mixed 查看已有分支
  • 当前分支带* 生成分支
  • 主分支用-a 切换到分支
  • 主分支用master 切换到新建分支 合并分支
  • 指定分支合并到当前分支 删除分支

GitHub使用

配置到github（来自github仓库创建过程）

如果是新建一个仓库：

  • -M将指定分支设定为主分支

如果是已有库：

  • 如果提示SSL证书问题，输入命令git config --global http.sslverify false
  • 输入GitHub账户信息

参与开源项目

  • 在GitHub项目中fork；或在本地git clone
  • git remote add upstream添加上游代码库链接
    查看代码库链接 创建分支并切换到新分支，同git switch -c 分支名 推送 第一次推送后要加参数指明push源和分支 如果显示冲突
