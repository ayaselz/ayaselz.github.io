Update Blog Content

以下命令可以把本地内容更新并部署到远端：

等价命令为hexo d -g。
在GitHub中，博客仓库右上角的缩略版本号前，提交状态若显示黄色的小圆点，则博客在更新中，提交的内容还未成功部署；若显示绿色的tick，则可以刷新博客页面查看更新的内容。

Change Theme (from GitHub)

主题推荐：

  • Fluid
  • Butterfly
  • NexT

Step 1: CLone

Code git clone 网络路径 本地路径

Example: Set Butterfly as the theme

Notes:

  • git clone 是克隆指令；

  • 网络路径 通常有.git结尾。不是网址，而是由Code处复制而来的URL；

  • themes/butterfly不是本地路径的唯一选择，只是存入themes文件夹更容易查找，因为搭建网页时Hexo已经在本地自动生成了这个名字的文件夹。同理，创建/butterfly文件夹是为了便于分类多个主题；

  • 网络路径和本地路径之间有一个空格。

Step 2: Configurate

  • 在本地内容中找到Hexo生成的_config.yml文件。这个文件包含了一系列设置参数；

  • 打开该文件（请勿使用Window自带的记事本，VSCode可以打开并编辑该类文件）；

  • 找到theme一栏，将冒号后面的内容改成主题的名字并保存。如下是butterfly的例子；

  • 更新部署到远端，再刷新个人博客就能实现（可能需要等待一段时间）。
